Frequently Asked Questions

Security & Privacy

Q: Can the server read my notes?
A: No. All encryption happens on your device (client-side) using AES-256-GCM before the data ever reaches the server. The server only stores encrypted blobs it cannot decrypt.
Q: Where is the decryption key stored?
A: The decryption key is embedded in the URL fragment (after the #), which clients never send to the server. Only the person with the full link can decrypt the note.
Q: Is FadNote really "zero knowledge"?
A: Yes. We use industry-standard AES-256-GCM encryption with PBKDF2 key derivation (100,000 iterations). The server has zero access to your plaintext or decryption keys.
Q: How do I know you're not lying about the encryption?
A: The entire server code is open source at https://github.com/easyFloyd/fadnote. You can audit it yourself or self-host.
Q: Is this really free?
A: Yes. The hosted version is my gift to the privacy-conscious. Self-host if you want. If it saves you time, consider buying me a coffee.

Self-Destruction & Expiration

Q: What does "self-destruct" mean?
A: After the first person opens the link and views the note, it's permanently deleted from the server. It cannot be viewed again.
Q: Can I view the note multiple times?
A: No. By design, notes are one-time read only. Make sure to save the content before closing the page.
Q: What if nobody opens the link?
A: Notes automatically expire after the TTL (Time-To-Live) you set. Default is 24 hour.
Q: Is there any backup or recovery?
A: No. Once viewed or expired, the note is gone forever. This is a feature, not a bug.

Usage

Q: What's the maximum note size?
A: 1MB per note. This is sufficient for text, code, API keys, passwords, and small documents.
Q: Can I send files or attachments?
A: Not currently. FadNote is text-only. For files, encrypt them locally first (e.g., with PGP) and share the encrypted text.
Q: Can I share the link via Slack/Email/WhatsApp?
A: Yes! The link works anywhere. Just remember: anyone with the link can view the note (once).
Q: What happens if I lose the link?
A: The link is the only way to access the note. If you lose it, the note is unrecoverable. Copy it immediately after creation.

Technical

Q: What encryption algorithm does FadNote use?
A: AES-256-GCM with PBKDF2 key derivation (100,000 iterations, SHA-256).
Q: Can I use my own FadNote server?
A: Yes! The code is open source. Set your FADNOTE_URL environment variable to point to your self-hosted instance.
Q: Does FadNote work offline?
A: No. Both the sender and recipient need an internet connection:
  • Sender: The CLI requires internet to POST the encrypted blob to the server
  • Recipient: Needs internet to fetch and decrypt the note from the server

Self-Hosting

Q: Can I run my own FadNote server?
A: Yes! The code is MIT licensed and available on GitHub. You can deploy via Docker.
Q: What are the requirements for self-hosting?
A: You have multiple deployment options:
  • Docker + Redis (recommended): Runs in a container with Redis for storage
  • Docker + Filesystem: Simpler setup using local file storage
  • Node.js directly: Run as a service with node (requires Redis or filesystem setup)
    • See the full self-hosting guide on github for detailed setup instructions.

Troubleshooting

Q: I get "Note not found or already viewed"
A: Either:
  1. The note was already opened (and deleted)
  2. The note expired (past TTL)
  3. The link was copied incorrectly
Q: The link doesn't work
A: Make sure you copied the entire URL including the part after #. The decryption key is in the URL fragment.
Q: Can I get a preview before sending?
A: No. For security, you must create the note to get the link. Test with a dummy message first if needed.
